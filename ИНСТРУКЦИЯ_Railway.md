# Бот отчётов ТОП-ТОЗА — запуск на Railway (24/7)

Схема: **GitHub** (хранит код) → **Railway** (запускает бота круглосуточно) →
бот читает **2 Google Таблицы** (9 км + Гульбута) и отвечает в **Telegram**.

Файлы проекта:
- `bot.py` — бот (две точки + общий итог)
- `requirements.txt` — библиотеки
- `Procfile` — команда запуска для Railway
- `runtime.txt` — версия Python
- `.gitignore` — что не загружать в GitHub (секреты)

---

## Часть A. Подготовка (один раз)

### A1. Токен бота
1. В Telegram откройте **@BotFather** → `/newbot` → имя и логин.
2. Скопируйте **токен** вида `1234567890:AAH...`

### A2. Две Google Таблицы
1. Залейте оба файла на https://drive.google.com и в каждом:
   **Файл → Сохранить как Google Таблицы**.
2. Из ссылки каждой таблицы скопируйте **ID** (между `/d/` и `/edit`):
   `.../spreadsheets/d/`**`ЭТО_ID`**`/edit`
   → получится два ID: для 9 км и для Гульбуты.
3. В обеих таблицах лист со сводкой должен называться **Дашборд**.

### A3. Ключ доступа Google (service-аккаунт)
1. https://console.cloud.google.com → **Создать проект**.
2. Включите **Google Sheets API** и **Google Drive API** (кнопка Enable на каждой).
3. **IAM и администрирование → Сервисные аккаунты → Создать**.
4. Откройте аккаунт → **Ключи → Добавить ключ → Создать ключ → JSON** (скачается файл).
5. В JSON найдите `"client_email": "...@...iam.gserviceaccount.com"` — скопируйте email.
6. В **обеих** Google Таблицах: **Открыть доступ** → вставьте этот email → роль **Читатель**.

## Часть B. GitHub (склад кода)

1. Зайдите на https://github.com → **New repository** → имя любое → **Create**.
2. Кнопка **Add file → Upload files** → загрузите файлы:
   `bot.py`, `requirements.txt`, `Procfile`, `runtime.txt`, `.gitignore`
   ⚠️ **Файл `service_account.json` НЕ загружайте** — ключ пойдёт в переменные Railway.
3. **Commit changes**.

---

## Часть C. Railway (сервер 24/7)

1. На https://railway.com в вашем проекте: **New → GitHub Repo** → выберите свой репозиторий.
   Railway сам увидит Python и `Procfile`.
2. Откройте сервис → вкладка **Variables → New Variable** и добавьте 4 переменные:

   | Имя переменной       | Значение                                            |
   |----------------------|-----------------------------------------------------|
   | `BOT_TOKEN`          | токен из шага A1                                     |
   | `SHEET_ID_KM9`       | ID таблицы «9 км» (A2)                               |
   | `SHEET_ID_GULBUTA`   | ID таблицы «Гульбута» (A2)                           |
   | `GOOGLE_CREDENTIALS` | **всё содержимое** файла JSON из шага A3 (целиком)   |

   > `GOOGLE_CREDENTIALS`: откройте скачанный JSON блокнотом, выделите всё (Ctrl+A),
   > скопируйте и вставьте в значение переменной как есть.

3. Railway автоматически перезапустит бота. Вкладка **Deployments / Logs** —
   там должна появиться строка: `Бот запущен. Ожидаю команды…`

## Часть D. Проверка
В Telegram напишите боту:
- `/otchet` — общий итог + обе точки
- `/km9` — только 9 км
- `/gulbuta` — только Гульбута
- `/myid` — узнать свой Telegram id

---

## Если захотите ограничить доступ
Чтобы отчёты видели только нужные люди: в `bot.py` в строке
`ALLOWED_USERS = []` впишите их Telegram id, например `ALLOWED_USERS = [123456789, 987654321]`
(каждый узнаёт свой id командой `/myid`). Загрузите изменённый `bot.py` в GitHub — Railway обновится сам.

## Частые ошибки
- **Бот молчит** → проверьте `BOT_TOKEN` в Variables и логи в Railway.
- **«Не получилось прочитать таблицу»** → не тот ID, лист не называется «Дашборд»,
  или email service-аккаунта не добавлен в доступ таблицы.
- **Ошибка JSON** → `GOOGLE_CREDENTIALS` вставлен не целиком.
